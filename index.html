Cascading Style Sheet Sprites

Cascading Style Sheets is a great way to spice up a website. We use CSS to style the layout of web pages through colors, fonts, spacing, and much more. One of many great things you can add to update your web page is CSS Sprites. Sprites are two dimensional bitmaps that are integrated into a larger scene. They were originally introduced back in 1975 as computer graphics that were even used in 2D video games. The term Sprite referred to fixed-sized objects composited together with a background by hardware.

Using sprites is a want and sometimes even a need. For example, a sprite may take less space than a regular large image. The difference would not be too big, but several images are loaded with a single HTTP request and the smallest difference really helps and sometimes is required. Another words, instead of including each image as a separate img file, it is much more memory to send them as a single image.

There are many alternatives to sprites however, they do come with their own pros and cons. Some of these similar ideas are SVGs, icon fonts, using and object-position, as well as data URIs. Sprites are still used everywhere. Mozilla Developer Network uses sprites to toggle their level navigation, Mailchimp uses sprites for their sidebar, and Mapbox uses an icon font (similar to sprites) for their smaller icons but they also use fairly high resolution sprites for social media and press logos. CSS sprites are a great way to implement some images without sacrificing lots of memory and it is still used to this day, in many, many creative ways.
